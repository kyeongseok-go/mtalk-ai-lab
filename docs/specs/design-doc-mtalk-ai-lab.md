# mTalk AI Lab — Design Document

## 한 줄 요약
LG CNS 사내 메신저 mTalk에 적용 가능한 7가지 AI 신기능을 통합 데모로 시연하는 웹 애플리케이션.

## 왜 만드는가? (Problem)
- LG CNS 경력직 지원 시 "이미 만들어봤습니다"라는 한 마디가 면접 임팩트를 극대화
- 사내 메신저는 하루 수백 개 메시지가 쌓이지만, 중요 정보 필터링은 100% 수동
- AI로 해결 가능한 반복 업무가 메신저 안에 집중되어 있음 (요약, 분류, 검색, 회의록)

## 누구를 위한 것인가? (User)
- **1차**: LG CNS 채용 면접관 (기술면접 + 임원면접)
- **2차**: 본인 포트폴리오 자산 (한화에어로스페이스 등 타사 지원에도 활용)
- **3차**: 실제 mTalk 사용자 (사내 메신저 사용하는 LG CNS 임직원)

## 무엇을 만드는가? (Solution)

### 7가지 AI 기능

| # | 기능명 | 핵심 가치 | Phase |
|---|--------|----------|-------|
| 1 | Smart Catch-up (AI 메시지 요약) | 안 읽은 87개 → 핵심 5줄 | Phase 1 |
| 2 | 시스템 알림 우선순위 분류 | 30개 알림 → 긴급/중요/참고 자동 분류 | Phase 1 |
| 3 | 회의록 자동 생성 | 채팅 50개 → 구조화된 회의록 | Phase 1 |
| 4 | 답변 초안 자동 생성 | 컨텍스트 기반 답변 제안 | Phase 2 |
| 5 | 의미 기반 메시지 검색 | 키워드 아닌 의미로 검색 | Phase 2 |
| 6 | 개인화 우선순위 학습 | 사용자 패턴 학습 시뮬레이션 | Phase 2 |
| 7 | 워크-라이프 모드 | 업무/개인 자동 분리 | Phase 2 |

### 토글 시스템 (핵심 차별점)
- 모든 기능에 ON/OFF 토글
- OFF = 기존 mTalk 경험 그대로
- ON = AI 기능 활성화
- **데모 효과**: 면접관이 직접 토글하며 AI 전/후 비교 체험

## 어떻게 만드는가? (Architecture)

### 기술 스택
- **Frontend**: Next.js 14 + TypeScript + Tailwind CSS + shadcn/ui
- **AI**: Claude API (Anthropic)
- **배포**: Cloudflare Pages + Edge Functions
- **디자인**: LG CNS 빨강 (#C8102E) 포인트 컬러

### 데이터 전략
- 실제 API 연동 없음 (사내 시스템 접근 불가)
- 현실적인 가짜 데이터로 시연
- 채팅방 5개, 인물 8명, 메시지 100+개

| 채팅방 | 성격 | 메시지 밀도 |
